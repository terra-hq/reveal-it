Reveal It By Terra

Reveal It is a sophisticated JavaScript module designed to animate web elements with finesse and precision. Leveraging the power of GSAP and its ScrollTrigger plugin, this library offers developers a versatile tool to create engaging, scroll-triggered animations that respond to user interactions and viewport changes.

Crafted with responsiveness in mind, Reveal It integrates seamlessly with dynamic web applications, providing a rich set of features to control animations based on viewport size. Whether you aim to enhance user experience on mobile devices or create captivating effects on desktop screens, Reveal It delivers with its adaptive and efficient animation capabilities.

  • Responsive Animation Controls: Tailor animations with breakpoint-specific configurations to ensure optimal display across all devices.
  • Advanced GSAP Integration:Advanced GSAP Integration: Utilize the full power of GSAP for smooth, high-performance animations with the added capability of ScrollTrigger for viewport-based triggers.
  • Mobile-First Design: With an option to enable or disable animations on mobile devices, Reveal It ensures that your site's performance and user experience are paramount.
  • Developer-Friendly: Built with ease of integration in mind, it requires minimal setup to get started, yet allows for deep customization for more complex scenarios.

Whether you're a front-end novice or an experienced developer, Reveal It provides the tools you need to bring your web elements to life. Dive into the documentation below to start creating animations that truly stand out.

Custom Reveal Animation

This example demonstrates creating a custom animation with specific parameters. You can specify options such as:

  • type: (outside of options) animation behaviour, 'from' by default, you can change it to 'to' or 'fromTo'.
  • duration: The length of time (in seconds) the animation runs.
  • opacity: The target opacity for the animated element, ranging from 0 (fully transparent) to 1 (fully opaque).
  • yoyo: A boolean value that, when set to true, causes the animation to go back and forth, effectively repeating the animation in reverse.
  • repeat: The number of times the animation should repeat. -1 indicates an infinite loop.
  • ease: Defines the rate of change of the animation. Examples include linear, power1.inOut, elastic.out, etc. GSAP offers a wide range of easing functions.
  • delay: Time in seconds before the animation starts after being triggered.
  • stagger: Used for animating multiple elements with a small delay between the start of each element's animation. Useful for sequenced animations.
  • scale, x, y, rotation: Transform properties for scaling, moving (along the x and y axes), and rotating the element.

Custom reveal animation with specific parameters...

                    
                    
// Import the RevealIt class
import RevealIt from "./RevealIt";

// Create a custom reveal animation
const customReveal = new RevealIt({
    element: document.querySelector(".js--reveal-it-custom"),
    type: "from",
    options: {
        duration: 2,
        opacity: 0,
        yoyo: false,
        repeat: 3
    }
});
document.getElementById('triggerCustomAnimation').addEventListener('click', 
function () {
    customReveal.getAnimation().restart();
});

The custom RevealIt instance allows you to define specific animation properties. This provides flexibility to tailor the animation to fit different scenarios or UI elements. We're also using .getAnimation().restart(); in the button to restart the animation.

Callbacks in RevealIt Animations

This example demonstrates how to use callbacks such as onStart and onComplete in RevealIt animations. These callbacks allow you to execute custom code at the beginning and end of the animation.

Element with RevealIt animation and callbacks...



    const cbExample = document.querySelector(".js--reveal-it-callback");
    const revealWithCallbacks = new RevealIt({
        element: cbExample,
        options: {
            duration: 1,
            opacity: 0,
            yoyo: false,
            repeat: 2,
            onStart: function () {
                cbExample.classList.remove('animation-completed');
                cbExample.classList.add('animation-started');
            },
            onComplete: function () {
                cbExample.classList.add('animation-completed');
                cbExample.classList.remove('animation-started');
            },
        }
    });
    
    document.getElementById('triggerAnimationWithCallbacks').addEventListener('click', function () {
        revealWithCallbacks.getAnimation().restart();
    });

The callbacks onStart and onComplete are useful for adding additional logic or effects at the start and end of the animation, respectively. They can be defined when creating a new instance of RevealIt.

ScrollTrigger with RevealIt Animations

This example demonstrates how to integrate ScrollTrigger into RevealIt animations. ScrollTrigger allows animations to be triggered based on scroll position, enhancing user interaction and engagement.

Scroll to see...

...the animation triggered by ScrollTrigger

    
    
    // Import the RevealIt class
    import RevealIt from "./RevealIt";
    
    // Create a RevealIt instance with ScrollTrigger
    const revealWithScrollTrigger = new RevealIt({
        element: document.querySelector(".js--reveal-it-scrolltrigger"),
        options: {
            duration: 1.5,
            opacity: 0,
            scrollTrigger: {
                start: "top center",
                end: "bottom top",
                toggleActions: "restart pause reverse pause",
            },
        },
    });

By specifying scrollTrigger options, you can control when the animation starts and ends based on the user's scroll position. The start, end, and toggleActions properties are key to configuring the scroll-based triggers.

Responsive Animation

This example demonstrates how to apply a continuous rotation animation to an element with the '.box2' class, adjusting its behavior based on the viewport size. Using GSAP's matchMedia, the animation rotates the element in one direction by default, reverses the rotation below the 'tabletm' breakpoint, and applies an additional scale transformation below the 'mobile' breakpoint. This approach enhances the interactive experience by providing visual feedback that adapts to different device sizes.

Resize your browser 

    
    
    // Define responsive animation options for the purple square
    const breakpointOptionsRotate = {
        defaultRotation: {
            minWidth: 'tabletm',
            animationOptions: {
                rotate: 360, 
                duration: 2, 
                repeat: -1, 
                ease: "none" 
            }
        },
        reverseRotation: {
            maxWidth: 'tabletm', 
            animationOptions: {
                rotate: -360, 
                duration: 2, 
                repeat: -1, 
                ease: "none" 
            }
        },
        reverseRotationScale: {
            maxWidth: 'mobile', 
            animationOptions: {
                scale: 2,
                duration: 2, 
                repeat: -1, 
                ease: "none" 
            }
        }
    };
    
    // Initialize the animation on the '.box2' element
    const box2Reveal = new RevealIt({
        element: document.querySelector('.box2'),
        animateOnMobile: true,
        breakpointOptions: breakpointOptionsRotate
    });

The RevealIt class, in conjunction with GSAP and its matchMedia feature, enables responsive and dynamic animations. By specifying different animation behaviors for various viewport sizes, content can be made more engaging and visually interactive across all devices.
